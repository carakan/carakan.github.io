mozilla 3 beta 1 no se deja con REST (Ruby on rails)

Carlos Ramos

November 26, 2007

Haciendo mi proyecto de grado/trabajo, trabajando e implementando diferentes tipos de respuestas bajo el diseño REST, cuando al estar probando Mozilla 3 B1, veía que no respondia a las peticiones vacias que no tenian el formato ej: /inscritos, me daba el error. Aparentemente no encontraba el valor del format en la linea de codigo:

respond_to do |format|

y según parece el valor de format no lo llena al hacer la petición en el navegador Mozilla 3 B1, así que tuve que desintalar este beta, debe de tener ciertos problemas o bugs de desarrollo(que macana el 2.0.9 me chupa la memoria). Lo mas curioso es que trate de arreglar este supuesto error del format, haciendo en cada request una filtración del parámetro params[:format], que por defecto sea html y si es Ajax (request.xhr?) no asignaba el valor .html. Perdi como tres horas tratando de arreglar este supuesto error :( Nota (18 agosto 2008): Vi que este error solo ocurren en la rama 1.2.x de Rails asi que solamente queda actualizar los proyectos a una rama superior, la actual es (2.1.0).
